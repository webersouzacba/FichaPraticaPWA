1. O que são operações de “Commit”, “Push” e “Pop” efetuadas num repositório?
  • Commmit - Cria uma nova versão dos arquivos no repositório local do GIT 
     
  • Push - Enviar os arquivos do repositório local (salvos em um Commmit) para o repositório remoto 
  • Pop - Aplica as mudanças do stash mais recente ao seu diretório de trabalho e removerá esse stash da lista de stashes 

 

  1. Quais sáo as vantagens de usar Branchs num repositório git? 
  • Permite a realização do desenvolvimento de versões diferentes em paralelo. 

 

  1. Nos Merges de conteúdos resultantes de um “Pop” podem resultar em conflitos. Em que consistem e como podem ser resolvidos esses conflitos?
O conflito de merge no Git ocorre quando há alterações conflitantes em um arquivo que está sendo combinado de diferentes branches ou stashes. No caso do comando Pop quando o há conflito de alterações de um determinado arquivo no Stash mais recente e no diretório de trabalho. 

 

  1. “Clone” e “Fork” são formas diferentes de replicar um repositório Git. Quais as diferenças entre eles? 
  • Clone cria uma cópia do repositório remoto do GitHub na máquina local. 
  • Fortk cria uma cópia do repositório remoto diretamente na conta remota do GitHub, permitindo evoluir o repositório clonado independente do repositório original.
