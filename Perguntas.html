Perguntas e Respostas sobre Git

  1. O que são operações de “Commit”, “Push” e “Pop” efetuadas num repositório?

    • Commit: Salva uma versão do que foi realizado no repositório local, sem atualizar o repositório Git remoto.
    • Push: Envia as mudanças salvas para o repositório remoto (GitHub ou outro).
    • Pop: Recupera as alterações salvas temporariamente com o uso do comando git stash.

  2. Quais são as vantagens de usar Branchs num repositório git?

    • Branchs: Permitem evoluir versões diferentes de um projeto sem interferência entre elas, facilitando o trabalho em equipe.
    • Quando necessário, pode ser realizado o Merge, que une as versões em uma única linha de evolução do projeto.

  3. Nos Merges de conteúdos resultantes de um “Pop” podem resultar em conflitos. Em que consistem e como podem ser resolvidos esses conflitos?

    Os conflitos acontecem quando mais de uma versão altera os mesmos trechos do código e o Git não sabe qual alteração manter.

    O Git informa onde ocorreram os conflitos e nós escolhemos qual versão irá prevalecer.

  4. “Clone” e “Fork” são formas diferentes de replicar um repositório Git. Quais as diferenças entre eles?

    • Clone: Ideal para trabalhar em projetos próprios ou da equipe. Faz uma cópia do repositório remoto na máquina local, mantendo vínculo com o original. Permite usar pull para atualizar e push para enviar mudanças.
    • Fork: Ideal para contribuir em projetos de terceiros. Cria uma cópia independente no seu GitHub. Permite alterações livres e envio de pull requests para o repositório original.
