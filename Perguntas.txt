1 - O que são operações de “Commit”, “Push” e “Pop” efetuadas num repositório?
    Resposta:
        Commit: Salva uma versão do que foi realizado no repositório local, sem atualizar o repositório Git;
        Push: Envia as mudanças salvas para o repositório remoto (GitHub ou outro);
        Pop: Recupera as alterações salvas temporariamente com a utilização do comando git stash.

2 – Quais sáo as vantagens de usar Branchs num repositório git?
    Resposta:
        Branchs: A utilização de Branchs permite evoluir versões diferentes de um projeto sem que uma versão interfira no conteudo da outra, permitindo o trabalho em equipe.
        Quando necessário pode ser ralizado o Merge que une as versão em uma única linah de evolução do projeto.

3 – Nos Merges de conteúdos resultantes de um “Pop” podem resultar em conflitos. Em que consistem e como podem ser resolvidos esses conflitos?
    Resposta:
        Os conflitos acontecem quando mais de uma versão alteram o(s) mesmo(s) trechos do código e o Git não sabe qual atlteração deve prevalecer. O Git informa onde ocorreram os conflitos e nós definimos qual versão irá prevalecer.

4 – “Clone” e “Fork” são formas diferentes de replicar um repositório Git. Quais as diferenças entre eles?
    Resposta:
        Clone:  É ideal para trabalho em projetos próprios ou da equipe em que participamos.
                Faz uma cópia do repositório remoto na máquina local mantendo um vínculo com o repositório remoto.
                Em um repositório clonado podemos puxar atualizações do original com o pull e enviar mudanças com o push.
        Fork:  É ideal para contribuir em projetos de outra pessoa, equipe ou insituição.
               Permite que sejam realizadas alterações no repositório local independente do repositório remoto original.
               Permite que sejam realizadas mudanças à vontade no repositório local e depois mandar um pull request para o repositório remoto original.
